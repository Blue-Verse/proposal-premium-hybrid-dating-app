핵심 요구사항 분석

  • 3채널 매칭 UX 분리: 외모(틴더형 스와이프, 얼굴 선공개) / 프로필(해시태그·텍스트 리스트, 스펙 선공개) / 커뮤니티(직군·관심사 익명 게시판) 채널을 독립 화면 흐름과 데이터 모델로 분리.
  • 차등 과금 + 이중 과금 방지: 채널별 정보 깊이에 따라 5/10/20 다이아 차감, 3채널 간 동일 프로필 열람 상태를 Redis 분산 락으로 동기화하여 동일 유저 중복 결제 방지.
  • 다중 서류 검증 + 즉시 파기: 신분증·자격증·소득금액증명원·자동차등록증을 KMS 봉투 암호화로 S3에 저장, 관리자 승인 직후 영구 삭제 스케줄러가 객체와 키를 동시에 파기.
  • 인앱 결제 + 다이아 지갑: Apple StoreKit 2 / Google Play Billing 영수증 서버 검증, 구매·환불·소비 트랜잭션의 멱등 처리 및 멀티 디바이스 지갑 동기화.
  • 프리미엄 호감 전송: 본인 연락처 첨부 호감 전송 시 고비용 다이아 차감 + 1회성 토큰 노출, 수신자 거부 시 연락처 자동 마스킹.
  • 관리자(Admin) 웹: 서류 심사·승인·파기, 어뷰징 신고·계정 정지, 가입자/매출 대시보드 (RBAC + 감사 로그).
  • 24시간 신규 회원 평가(Rating): 가입 직후 24h 동안 기존 회원 평가 노출, 평가 점수에 기반한 노출 가중치 산정.
  • Phase 2 확장 대비: 행동 로그·해시태그를 정형화하여 적재해 추후 NLP 기반 AI 큐레이션 매칭 도입 가능한 데이터 모델 설계.

제안 포인트

① 매칭·프로필·메시징 플랫폼 직접 수행 경험 — 비즈니스 매칭 플랫폼(Connectin)을 Flutter + NestJS 풀스택으로 3개월 만에 베트남 양대 마켓 출시한 경험을 그대로 데이팅 도메인에 전이합니다. 명함 매칭에서 검증된 BLE 근거리 탐색·실시간 프로필 동기화·E2E 암호화 메시징 패턴이 본 프로젝트의 3채널 매칭과 1:1 대화 요청 기능에 직접 적용됩니다.

② 인앱결제 + 게이미피케이션 다이아 시스템 검증된 패턴 — Flutter B2C 앱(DayStarter, Fortune App)에서 Apple/Google 인앱결제 + 자체 다이아·포인트 경제 + 멀티 디바이스 지갑 동기화를 운영해 본 노하우를 가지고 있어, 차등 과금 알고리즘과 이중 과금 방지 로직을 안정적으로 설계할 수 있습니다.

③ 민감 정보 보안 아키텍처 설계 경험 — VC 펀드 관리 플랫폼(Series-B)에서 NICE KYC, 5계층 보안, DLP, GeoIP 기반 접근 제어를 구축한 경험으로 KMS 봉투 암호화·즉시 파기 스케줄러·감사 로그를 금융권 수준으로 설계합니다.

④ 양대 마켓 동시 런칭 노하우 — Flutter 단일 코드베이스로 iOS·Android 동시 출시(Calendar Share, DayStarter, Fortune App, Connectin) 경험으로 Apple App Review, Google Play 정책 리젝 사유(데이팅 카테고리 특수 정책 포함)를 사전 점검합니다.

⑤ 1순위(금액) 부합 + 2순위(완성도) 보장 — 본 견적은 클라이언트 예산 상한 대비 효율적 구성으로 책정하되, 검증된 플랫폼 패턴 재활용으로 산출물 완성도를 동급 견적 대비 우위로 보장합니다.

본 PRD는 공고 명시 사항을 기반으로 1차 정리한 초안이며, 킥오프 워크숍에서 클라이언트와 함께 확정합니다.

1. 프로젝트 배경 및 목적

경제적 여유와 뚜렷한 라이프스타일을 가진 20대 후반~30대 중반 타겟층을 위한 프리미엄 데이팅 플랫폼을 구축합니다. 단순 스펙 나열을 넘어 외모·프로필·커뮤니티 3채널을 분리해 사용자가 원하는 방식으로 매칭을 시도하게 하고, 채널별 정보 깊이에 따라 다이아(가상 재화)가 차등 차감되는 하이브리드 과금 모델을 구현합니다. 벤치마킹 모델은 스카이피플이며, 신뢰도 높은 유저 풀을 위해 다중 서류 검증을 거치되 민감 서류는 검증 직후 즉시 파기하는 보안 아키텍처를 핵심 차별점으로 합니다.

3. 기능 요구사항

3.1 가입 및 인증 시스템

  • [필수] 휴대폰 본인인증 연동 (NICE / KCB)
  • [필수] 다중 서류 업로드 (신분증, 자격증, 소득금액증명원, 자동차 등록증)
  • [필수] 업로드 서류 KMS 봉투 암호화 저장 (S3 + KMS Customer Master Key)
  • [필수] 관리자 승인 직후 즉시 파기 스케줄러 (S3 객체 + KMS 데이터 키 동시 삭제, 감사 로그 보존)
  • [필수] 신규 가입자 24시간 회원 평가(Rating) 시스템 + 평가 가중치 노출 알고리즘
  • [필수] 가입 거절 시 사유 안내 및 재가입 제한 정책

3.2 3채널 매칭 UI/UX

  • [필수] 외모 매칭: 틴더형 카드 스와이프 (좋아요/패스), 얼굴 선공개, 상세 스펙 블라인드 처리
  • [필수] 프로필 매칭: 해시태그/텍스트 리스트 UI, 상세 스펙 선공개, 얼굴 사진 블라인드 처리
  • [필수] 커뮤니티 매칭: 직군/관심사 기반 익명 게시판, 게시글/댓글/좋아요, 1:1 대화 요청
  • [필수] 채널 간 동일 프로필 열람 상태 실시간 동기화 (Redis Pub/Sub)
  • [필수] 차단·신고 기능 채널 전반 적용

3.3 차등 과금 및 인앱 결제

  • [필수] Apple App Store 인앱결제 (StoreKit 2) + Google Play Billing 연동
  • [필수] 다이아 충전 패키지 (Tier별 상품 구성, 추후 Admin에서 단가 변경 가능)
  • [필수] 채널별 차등 차감 알고리즘 (외모 5다이아, 프로필 10다이아, 커뮤니티 20다이아 등 정책 가능)
  • [필수] 이중 과금 방지: 동일 프로필 24시간 내 중복 열람 시 추가 차감 없음 (Redis 분산 락)
  • [필수] 영수증 서버 검증 (App Store / Play Store) + 환불·취소 트랜잭션 멱등 처리
  • [필수] 다이아 지갑 잔액·거래 내역 조회 + 멀티 디바이스 동기화

3.4 프리미엄 호감 전송

  • [필수] 호감 전송 시 본인 실제 연락처 다이렉트 첨부 + 고비용 다이아 차감
  • [필수] 수신자 수락 시 연락처 노출, 거부 시 자동 마스킹 + 다이아 일부 환원 정책 (협의)
  • [필수] 호감 전송/수신 알림 (FCM Push)

3.5 관리자(Admin) 페이지

  • [필수] 서류 심사 큐 (승인/반려, 사유 메모, 즉시 파기 제어)
  • [필수] 어뷰징 신고 관리 (신고 리스트, 처리 상태, 계정 정지/해제)
  • [필수] 가입자 통계 (일/주/월 가입 추이, 활동 회원 수)
  • [필수] 다이아 결제·소모 매출 대시보드 (채널별 소비 패턴, ARPU)
  • [필수] 관리자 RBAC + 모든 민감 작업 감사 로그 (Audit Log)

B2B 네트워킹 / 매칭 / 베트남 시장 · 2025.05–2025.08 (3개월)

프로젝트 개요

물리적 명함 교환의 한계를 해결하기 위해 OCR 스캐닝, E2E 암호화 메시징, BLE/GPS 근거리 탐색을 결합한 디지털 명함·네트워킹 플랫폼. 비즈니스 전문가는 디지털 명함을 만들고 근처 전문가를 실시간으로 발견·연결합니다.

핵심 기능

  • Signal Protocol E2E 암호화 메시징 (서버 운영자조차 메시지 미열람)
  • BLE + GPS 이중 모드 근거리 사용자 탐색 (오프라인 매칭)
  • OCR 명함 스캐닝 (자동 디지털화)
  • 소셜 피드 + 1:1 메시징 + 비즈니스 네트워킹
  • 관리자 패널 (RBAC, IP 화이트리스트, 대량 사용자 관리)
  • 12개 마이크로서비스 + 63 API 엔드포인트 + 41 모바일 기능 모듈

본 프로젝트와의 유사점

① 매칭 플랫폼 도메인 직접 경험 — 프로필 기반 발견·매칭·1:1 대화 요청 흐름이 본 프로젝트의 3채널 매칭과 구조적으로 동일합니다. ② Flutter + NestJS 풀스택 — 동일 기술 스택으로 양대 마켓 출시. ③ 보안 매칭 — Signal Protocol 수준의 암호화 설계 경험으로 KMS 봉투 암호화·즉시 파기 아키텍처를 안정적으로 적용 가능. ④ 3개월 풀스택 딜리버리 — 100일 일정의 실현 근거.

B2C 모바일 앱 / 라이프스타일 · 2024.07–2024.09 (3개월)

프로젝트 개요

사주 분석 엔진 연동, 행운 번호 추천, 궁합 분석, 포인트·데일리 체크인 리워드, 자체 광고 SDK를 결합한 Flutter B2C 앱. 기획→디자인→개발→런칭까지 100% 단독 수행하여 양대 마켓 출시.

핵심 기능

  • 사주(LF3) 분석 엔진 연동, 운세 결과 화면
  • 포인트 & 데일리 체크인 리워드 (게이미피케이션)
  • 자체 광고 SDK (flutter_ads) — 수익화 모델
  • Google / Apple SSO 소셜 로그인
  • 운세 결과 스크린샷 공유 기능
  • BLoC/Cubit + Clean Architecture (305 Dart 파일, ~16K LOC)

본 프로젝트와의 유사점

① Flutter B2C 풀사이클 3개월 딜리버리 — 본 프로젝트와 동일한 기획+디자인+개발 단독 수행 + 100일 양대 마켓 출시 패턴. ② 게이미피케이션 + 포인트 경제 — 다이아 지갑·차등 차감과 동일한 가상 재화 경제 설계 노하우. ③ 양대 마켓 심사 통과 — Apple/Google 심사 사유 사전 점검 경험.

DayStarter (Sleep-to-Earn 게이미피케이션 앱)

B2C 모바일 앱 / 헬스케어 · 2022.06–현재 (약 3년 운영)

프로젝트 개요

센서 기반 수면 검증과 7단계 루트박스 이코노미를 결합한 Sleep-to-Earn 게이미피케이션 앱. 행동 변화 게이미피케이션 + 가상 재화 경제 + Spine 2D 캐릭터 시스템을 운영.

핵심 기능

  • 센서 기반 수면 검증 (ML Kit, 조도, 배터리 신호 융합)
  • 7단계 루트박스 가상 재화 이코노미
  • Spine 2D 캐릭터 시스템 (게임형 UX)
  • 5개 레포 + 37 DB 모델 + ~50K LOC + 약 3년 지속 운영
  • 인앱결제 + 광고 + 자체 포인트 경제 운영 노하우

본 프로젝트와의 유사점

① 가상 재화 경제 장기 운영 경험 — 다이아 지갑·차등 차감·이중 차감 방지 로직과 동일한 도메인. 3년 간 운영하며 결제·환불·악용 시나리오 전반에 대한 운영 노하우 보유. ② Flutter + NestJS + PostgreSQL — 본 프로젝트 권장 스택과 동일. ③ 행동 데이터 정형화 적재 — Phase 2 NLP 큐레이션을 위한 행동 로그·해시태그 데이터 모델 설계 노하우 직접 활용 가능.

기술 스택

Flutter NestJS TypeScript PostgreSQL Firebase ML Kit Spine 2D In-App Purchase
